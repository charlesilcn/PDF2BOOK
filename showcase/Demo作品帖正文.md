## 1. Demo 简介

**是什么：** PDF2BOOK 是一个 AI 驱动的自动排版工具——用户只需把扫描版 PDF 丢进来，说一句「转成 EPUB」，AI 就像一位编辑一样全自动完成：OCR 识别 → 判断页面类型 → 提取元数据 → 校对错字 → 推断章节结构 → 生成带封面、有目录、分章节、Kindle 友好的 EPUB。全程零干预，用户给 PDF，拿 EPUB。

**面向谁：** 电子书爱好者、Kindle/文石等电纸书用户、学生和研究者（大量扫描版学术资料、绝版参考书）、收藏老旧扫描书籍的读书人。

**核心功能：**

1. **AI 全自动 9 步流程** — 页面类型识别、元数据提取、OCR 智能校对、排版参数推断，全是 AI 智能决策，不是规则匹配。用户不需要判断哪页是封面、不需要手填书名作者、不需要逐字校对、不需要懂排版参数。
2. **Trae Skill 一句话触发** — 在 Trae 中说「把 XX.pdf 转成 EPUB」，AI agent 自动完成全流程。也可用 CLI 命令行批量处理。
3. **真实场景验证** — 已用 162 页中文扫描书《希腊神话故事》完整验证，AI 自动分出 42 个故事章节，每章独立成页，目录可跳转，封面/版权/目录页自动识别并特殊处理（封面渲染为图片，版权页提取元数据，目录页转为可点击链接），元数据自动提取。

## 2. Demo 创作思路

**灵感来源：** 我自己就是一个 Kindle 爱好者，收藏了上百本扫描版文学书。每次想把它们转到 Kindle 上，要么字体小到要放大镜，要么用转换工具出来是一坨没有结构的"文字流"，找故事、翻章节全靠滑屏。我意识到，真正的电子书排版需要的不是 OCR（识别文字），而是一个"编辑"——判断哪页是封面、哪页是目录、书名叫什么、作者是谁、每个故事从哪里开始。这件事传统工具做不了，但 AI 可以。

**想解决的问题：** 很多人手上有大量扫描版 PDF 图书（文学经典、专业书籍、绝版资料），想放到 Kindle 上阅读，但：
- 扫描版 PDF 在 Kindle 6 寸屏上字体太小，根本看不清
- 传统转换工具（如 Calibre）只能"机械搬运"——输出一整本无结构的长页面，没有目录、没有分章、封面丢失，还是没法读
- 想做好排版需要手动折腾数小时（判断页面类型、手填书名作者、校对 OCR 错字、调分章参数），技术门槛高，普通用户根本搞不定
- 现有 OCR 工具识别完文字就结束，不会"理解"内容结构，用户还得自己整理

**为什么做这个方向：** 核心判断是——电子书排版的瓶颈不在 OCR（文字识别已经成熟），而在"编辑决策"（理解内容结构）。传统工具把所有决策都甩给用户，而 AI 可以承担这些决策。我的取舍是：不追求支持所有 PDF 类型，而是把"扫描版文学书 → Kindle EPUB"这一个场景做到极致——全程零干预，用户只管给 PDF 和拿 EPUB。

## 4. TRAE 实践过程

### 开发流程

整个项目基于 **TRAE IDE** 开发完成，全程使用 AI agent 辅助开发。开发流程如下：

**阶段一：需求拆解与架构设计**
用 TRAE 梳理扫描版 PDF 转 EPUB 的完整流程，拆解出两阶段架构：OCR（昂贵，缓存）→ 后处理（廉价，可重跑）。AI 帮我分析了 PaddleOCR PP-StructureV3 的 JSON 字段映射，设计了可插拔的 OCR 后端抽象。

**阶段二：核心流水线实现**
用 TRAE 实现了 OCR 后端集成、页面分类器（规则化识别封面/版权/目录/正文）、CIP 元数据提取（GB/T 12451 标准）、置信度三级过滤、跨页段落合并、标题层级推断、图片裁剪等模块。每个模块配单元测试。

**阶段三：AI 审查流水线**
用 TRAE 设计了 Markdown-based AI 审查流水线——AI 直接审查 book.md 完成所有最终阶段任务：OCR 校对、标题修正、元数据校验、段落排版、章节结构验证、目录链接化。通过 `::: {.toc-list}` 哨兵实现幂等性。

**阶段四：Trae Skill 集成**
用 TRAE 编写了 `.trae/skills/pdf2book/SKILL.md`，让 AI agent 通过自然语言触发完整流程。Skill 路径下 AI agent 自身承担所有"需要理解内容"的决策，无需外部 LLM API key。

**阶段五：工程化与测试**
用 TRAE 完成了三文件夹零配置布局（inbox/library/workspace）、批量并行处理、断点续作、配置驱动、642 个单元测试。最后用 162 页真实扫描书完整验证。

- **AI 是决策者，不是工具调用者**：页面类型识别、元数据提取、OCR 校对、排版参数推断，全是 AI 智能决策

- **Trae 原生 Skill**：一句话触发全流程，AI 自动完成 9 步全自动流程
- **真实场景验证**：已用 162 页中文扫描书《希腊神话故事》完整验证，自动分出 42 个故事章节
